# 全球MEG物流图（含表格+世界地图）设计文档

## 1. 目标与范围

目标：基于既有两张模板图的视觉体系，新增一张“全球MEG物流”图，包含模板要求的两个核心模块：

- 区域汇总表格
- 世界地图物流流向示意

范围限定：

- 以定性为主、定量为辅，不追求海关口径精算精度
- 使用 2024 年口径的“区域级大致流向”
- 输出 SVG + PNG，并保留可重复生成脚本

## 2. 已确认设计决策

### 2.1 画面结构

- 横版画布，延续既有模板比例与灰底风格
- 顶部标题线（灰线 + 红色短线）与右上角标识（CIEC）
- 左上：区域净出口/净进口表格
- 主体下方：世界地图 + 跨区域物流箭头
- 底部：数据来源与口径说明

### 2.2 区域划分

采用 8 区域统一口径：

- 北美
- 西欧
- 中东
- 东北亚
- 东南亚
- 南亚
- 南美
- 非洲

### 2.3 数据表达规则

- 表格字段：`区域`、`角色`（净出口/净进口）、`大致规模`（Mt，取整）、`主要流向/来源`
- 地图仅展示主干流向（控制在 8-12 条）
- 数值按量级分级展示（如 1-3、3-6、6+ Mt）
- 箭头粗细对应流量等级，颜色对应来源区域
- 进口终点用浅色高亮圈标识

### 2.4 文案与口径说明

- 提供 2-3 条一句话结论（如“中东为核心外供区、亚洲为主要吸纳区”）
- 脚注说明：基于 Reuters / 公开行业资讯 / AI 辅助整理，定性展示为主，非海关精算口径

### 2.5 输出与命名

输出目录：`results/pictures`

- `meg-global-logistics.svg`
- `meg-global-logistics.png`
- `render_meg_global_logistics.py`

## 3. 备选方案评估（已完成选择）

方案 A：仅公开资讯 + 行业常识，直接一次成图（最快，追溯性弱）

方案 B（已选）：1 个 AI 源 + 1 个权威新闻源提炼区域流向后出图（速度与依据平衡最佳）

方案 C：完全机构报告口径（一致性高，但耗时长、数据时滞）

最终选择：方案 B。

## 4. 信息架构与数据流

1. 查询公开信息（优先英文/全球来源）形成区域级流向草表。
2. 将草表转换为图形输入：
   - 区域属性表（角色/规模/来源或去向）
   - 物流边表（起点、终点、等级、注释）
3. 渲染程序按既有模板风格出图：先背景和标题，再表格，再地图与箭头，再脚注。
4. 导出 SVG（主稿）与 PNG（汇报稿）。

## 5. 组件与职责

- 脚本文件：`results/pictures/render_meg_global_logistics.py`
  - 负责数据定义、布局、样式、渲染、导出
- 结果文件：
  - `results/pictures/meg-global-logistics.svg`
  - `results/pictures/meg-global-logistics.png`

采用单脚本生成，便于后续替换数据和快速微调版式。

## 6. 错误处理与版式风险

主要风险与处理：

- 箭头重叠：通过分层弧线、终点错位和等级筛选降低拥堵
- 标签溢出：限制文本长度、对超长字段换行、关键文本避让边缘
- 表格拥挤：固定列宽与字体等级，必要时缩短“主要流向/来源”描述
- 底部冲突：脚注与结论留足垂直间距，防止与图例重叠

## 7. 验收标准

1. 同时包含表格和世界地图两个核心模块。
2. 视觉风格与既有图一致（背景、标题线、配色层级、脚注样式）。
3. 无可见重叠与溢出（箭头不压标题/脚注，表格列对齐清晰）。
4. 文件在 `results/pictures` 可直接用于汇报。

## 8. 测试与验证方式

- 运行脚本生成 SVG/PNG，检查是否正常输出。
- 人工检查三项：
  - 模块完整性（表格+地图）
  - 视觉一致性（模板风格）
  - 排版可读性（无重叠、无截断）

## 9. 非目标（避免范围蔓延）

- 不做国家级精细贸易流矩阵
- 不做交互式地图
- 不引入额外外部可视化框架（保持与当前工作流一致）
